How AI Context Works

Why does the chat "forget" things? Why does it sometimes edit the wrong file? This guide explains how context windows work and how to get better results.

🧠

What IS Context?

When you talk to an AI (ChatGPT, Copilot, Claude), it doesn't have a memory in the human sense. Instead, every message you send — and every response the AI gives — is packed into a context window.

Think of it like a whiteboard with a fixed size. Every new message is written on the whiteboard. When the whiteboard is full, the oldest content gets erased from the top.

4K–8K

1M+

1 token ≈ 4 characters in English, ≈ 2–3 characters in Polish

😤

Why Does It "Forget" Things?

🗑️ Token Overflow

When conversation exceeds the context window, the earliest messages are silently dropped. The AI literally can't see them anymore.

🌫️ Attention Dilution

Even within the window, AI models have limited attention. Info in the middle of a long conversation gets less focus than the beginning or end.

📂 Wrong Project?

If you reference multiple apps/files, the AI can confuse which project you mean — especially if early messages about Project A got truncated.

🔄 No Persistent Memory

Each new chat session starts from zero. Previous conversations are not carried over unless explicitly pasted back in.

📊 The "Lost in the Middle" Effect

Research shows AI models recall the beginning and end of context best, but information in the middle is often "lost." This is called the Lost in the Middle phenomenon (Liu et al., 2023).

← Start (high recall) Middle (low recall) → End (high recall) →

How VS Code Copilot Builds Context

When you use Copilot in VS Code (Agent mode), it's smarter than plain ChatGPT — it actively gathers context using tools:

📁

File Reading

The agent reads specific files/lines from your workspace into its context window.

🔍

Semantic Search

It searches your codebase for relevant code snippets using meaning-based search.

🖥️

Terminal Commands

It runs commands (curl, grep, git, etc.) and reads their output into context.

📋

Conversation Summary

When a conversation gets very long, VS Code automatically creates a conversation-summary that compresses earlier context into a structured recap.

⚠️ But It Still Has Limits!

Even with all these tools, the context window is finite. When you have a very long conversation spanning many projects, the agent may lose track of which project is which, or forget specific file paths.

Best Practices for Building Context

1

Start New Chats for New Tasks

Don't continue a 2-hour conversation for a new topic. Fresh chat = clean context = better accuracy.

2

Be Explicit About Which Project/File

Instead of "fix the bug", say "in KanGo/app.py, fix the bug in the render() function." Absolute paths reduce ambiguity.

3

Write Clear Prompts in Tasks

In KanGo, use the Prompt field to write exactly what the AI should do. Include file paths, expected behavior, and constraints.

4

Use @workspace and @file References

In Copilot Chat, use @workspace to search across all files, or #file:path to explicitly include a file.

5

Front-Load Important Info

Put the most critical instructions at the beginning of your message. Due to the "lost in the middle" effect, the start and end get the most attention.

6

Batch Similar Work Together

If you need 5 changes to KanGo, do them all in one session. Switching between projects in one chat leads to confusion.

7

Keep a RUNBOOK / README

Maintain a RUNBOOK.md in each project. The AI can read it to quickly understand architecture, deploy commands, and constraints.

Common Mistakes to Avoid

✗

Mixing many projects in one conversation. After 30+ messages, the AI can't tell KanGo from WQD from StratosX.

✗

Vague references. "Fix that bug" → which bug? In which file? Be specific.

✗

Expecting memory across sessions. The AI doesn't remember yesterday's chat. Paste key decisions or use a runbook.

✗

Not providing constraints. "Deploy this" without specifying project, region, or service name leads to guessing.

✗

Running background + foreground in same terminal. This aborts running deploys — use separate processes.

TL;DR

AI context is a fixed-size window. It forgets because old messages fall off. Be specific, start fresh chats, write good prompts, and keep runbooks. The better your input, the better the output.
