6 Copilot Autopilot Workflow

1️⃣
Pick a task from "For Production"
GET /api/tasks → filter bucket == "for-production", sort by priority (critical first), pick one that has a prompt filled in.
2️⃣
Set in_progress = true
PUT /api/tasks/<id> with {"in_progress": true}. This triggers blue glow + pulse animation on the board (visible to human watching).
3️⃣
Execute the prompt
Read the prompt field. Open the relevant app directory (app_directory). Execute the work — code changes, file generation, etc.
4️⃣
Write execution log in comment
PUT /api/tasks/<id> with {"comment": "✅ What was done:\n- detail 1\n- detail 2\nDeploy: revision-name"}. Always start with ✅ if successful.
5️⃣
Move to "For Tests — Confirm"
PUT /api/tasks/<id> with {"bucket": "for-tests-confirm", "in_progress": false, "done": true}. The card moves and gets green glow.
6️⃣
Pick next task
Repeat from step 1. Human reviews "For Tests" tasks and moves them to Delivered → Done.

8 Conventions & Rules

  • ●
    System UI in English — all labels, buttons, messages, error text must be English
  • ●
    Comments start with ✅ for completed tasks — this triggers the green glow on the card
  • ●
    No prompt = red border — tasks without a prompt field get a red left border as visual indicator
  • ●
    Estimated time = Vibe Coding speed — estimate based on AI agent work, not manual human effort
  • ●
    HTML outputs go to Files — generated reports/analyses get uploaded to GCS via /api/files and linked in comment
  • ●
    One task at a time — set in_progress on ONE task, complete it, move to next. Don't work on all for-production tasks simultaneously
  • ●
    UI aesthetic = Tailwind + Outfit font — clean, modern, TailAdmin-inspired. SVG icons, no emojis in system UI (emojis OK in bucket labels)
  • ●
    Live polling — board auto-refreshes every 5 seconds. Changes via API appear on screen without manual reload
