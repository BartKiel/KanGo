7. 💾 Backupy

Automatyczne (Cloud Scheduler)

  • KanGo: codziennie o 3:00 CET → kango-backups-daily/YYYY-MM-DD/
  • StratosX: codziennie o 3:00 CET → stratosx-backups-daily/YYYY-MM-DD/
  • Format: pełna kopia GCS bucket (wszystkie pliki JSON + uploads)
  • Retencja: 30 dni (starsze ręcznie usuwane)

Ręczne odtworzenie z backupu

  1. Zaloguj się do GCP Console → Storage
  2. Znajdź bucket backupowy (np. kango-backups-daily)
  3. Wybierz folder z datą (np. 2026-03-10/)
  4. Skopiuj pliki z backupu do produkcyjnego bucketa:
    gsutil -m cp -r gs://kango-backups-daily/2026-03-10/* gs://kango-tasks-esj-bk/

Kod źródłowy

Cały kod jest na GitHub. Wystarczy git clone i gcloud run deploy.

9. 🔄 Procedura Odzyskiwania (krok po kroku)

Scenariusz: Bart jest niedostępny, system wymaga interwencji

  1. Loginy: Wejdź do 1Password (vault "WebQuest Shared") — tam są WSZYSTKIE hasła
  2. GCP Console: Zaloguj się na bart@webquestdigital.com → console.cloud.google.com
  3. Jeśli app nie działa: Cloud Run → service → Revisions → przywróć poprzednią rewizję klikając "Route 100% traffic"
  4. Jeśli dane zniknęły: Cloud Storage → bucket backupowy → skopiuj najnowszy backup (patrz sekcja 7)
  5. Jeśli trzeba zdeployować nowy kod: Zainstaluj gcloud CLI, sklonuj repo z GitHub, uruchom gcloud run deploy
  6. Jeśli domena nie działa: Sprawdź DNS w panelu rejestratora, odnów domenę jeśli wygasła
  7. Klient pilnie potrzebuje czegoś: StratosX działa autonomicznie — klienci mogą się logować i korzystać bez naszej interwencji
